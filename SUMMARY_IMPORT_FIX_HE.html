דוקומנטציה טכנית לצוות העמותה

⚠️ הבעיה שתוקנה

בעיה ביצור:
כאשר יובאו חונכים וחניכים מקובץ Excel, וברשומה היתה המידע "יש חניך" (או "יש חונך" לחניך), המערכת הייתה משאירה את הרשומה בסטטוס "יש חניך" גם לאחר שהחונכות היתה מחוקה. זה יצר אי-עקביות - החונך עדיין נראה "תופוס" גם שלא היה לו חניך.

📋 דוגמה של הבעיה:

1️⃣ קובץ Excel מכיל חונך עם סטטוס "יש חניך"
2️⃣ המערכת יובאה את החונך עם סטטוס "יש חניך" (לפי הקובץ)
3️⃣ אדמין מוחק את החונכות
❌ בעיה: החונך עדיין מראה "יש חניך" - סטטוס שגוי!

✅ השינויים שבוצעו

1️⃣ הוספת שדות מעקב בטבלאות (SQL)

קובץ: add_imported_flags.sql

ALTER TABLE childsmile_app_tutors ADD COLUMN is_t_imported BOOLEAN DEFAULT FALSE;
ALTER TABLE childsmile_app_children ADD COLUMN is_c_imported BOOLEAN DEFAULT FALSE;

מה זה עושה:

  • ✓ is_t_imported - שדה בוליאני בטבלת החונכים - מסמן אם החונך יובא מקובץ Excel
  • ✓ is_c_imported - שדה בוליאני בטבלת החניכים - מסמן אם החניך יובא מקובץ Excel
  • ✓ DEFAULT FALSE - רק רשומות מיובאות יהיו TRUE

2️⃣ הוספת סימון ייבוא בלוגיקת הייבוא

קובץ: views_volunteer.py - הוספת is_t_imported=True בשלוש מקומות

Tutors.objects.create(
    id_id=signedup.id,
    staff=staff,
    tutorship_status="יש_חניך",        # בדיוק כמו קודם
    tutor_email=email,
    preferences=tutor_preferences,
    is_t_imported=True,                # ✅ זה בלבד הוסף!
)

מה השתנה:

הוסיפו: שדה is_t_imported=True לכל Tutors שמיובא

לא שינו: הלוגיקה של הייבוא נשארת בדיוק אותו הדבר - כל הסטטוסים מהקובץ מיובאים כמו שהם

זה בלבד הוסף: סימון לצוותה שהרשומה מיובאת, כדי שנוכל להתייחס אליה בצורה מיוחדת כשמוחקים חונכות

3️⃣ שינוי לוגיקת מחיקת חונכות

קובץ: tutorship_views.py - שורות 1019-1042

# אם החונך יובא מקובץ
if tutor.is_t_imported:
    tutor.tutorship_status = "אין_חניך"   # ✅ חזור ל "אין חניך"
    tutor.is_t_imported = False            # ✅ כבר לא מיובא
    tutor.save()

# אם החניך יובא מקובץ (ואין עוד חונכות)
if not other_child_tutorships:
    if child.is_c_imported:
        child.tutoring_status = "אין_חונך"  # ✅ חזור ל "אין חונך"
        child.is_c_imported = False         # ✅ כבר לא מיובא
        child.save()

מה זה עושה:

  • כאשר מוחקים חונכות של חונך מיובא:
    • סטטוס החונך חוזר ל "אין_חניך" (לא "יש חניך")
    • הדגל is_t_imported הופך ל FALSE (עכשיו החונך מטופל כרשומה רגילה)
  • כאשר מוחקים חונכות של חניך מיובא:
    • סטטוס החניך חוזר ל "אין_חונך" (לא "יש חונך")
    • הדגל is_c_imported הופך ל FALSE
    • רק אם זאת החונכות האחרונה (יכול להיות עוד חונכים)

is_t_imported = FALSE ✅ (כבר לא מיובא - זה מה שצפוי לחונך מיובא)

🎯 למה זה חשוב?

בעיה פתרון תוצאה
קובץ אומר "יש חניך" אבל אנחנו לא יודעים שהרשומה מיובאת סימון עם is_t_imported=TRUE ✅ אנחנו יודעים מה היה מיובא
מחיקת חונכות של חונך מיובא משאירה אותו בסטטוס הלא נכון בדיקת הדגל is_t_imported בעת מחיקה ✅ החונך חוזר לסטטוס נכון אחרי מחיקה
חניך מיובא נשאר "יש חונך" אחרי מחיקת כל החניכויות בדיקת הדגל is_c_imported בעת מחיקה ✅ החניך חוזר לסטטוס נכון אחרי מחיקה

🧪 נקודות חשובות לבדיקה

📊 אחרי הוספת העמודות SQL: 
SELECT COUNT(*) FROM childsmile_app_tutors WHERE is_t_imported = TRUE;
SELECT COUNT(*) FROM childsmile_app_children WHERE is_c_imported = TRUE;

בדיקת ייבוא:

  • יבאו חונך עם סטטוס "יש חניך" בקובץ
  • בדקו בטבלה: tutorship_status צריך להיות "אין_חניך"
  • בדקו בטבלה: is_t_imported צריך להיות TRUE

בדיקת מחיקה:

  • יצרו חונכות להחונך המיובא
  • מחקו את החונכות
  • בדקו בטבלה: tutorship_status צריך להיות "אין_חניך" (לא "יש חניך")
  • בדקו בטבלה: is_t_imported צריך להיות FALSE

🎉 תוצאה סופית

לא עוד בעיית "יש חניך"!

כאשר מחקנים חונכות, סטטוסי החונך/ה חוזרים לנכון ולא נשארים "תפוסים" בטעות.

🧹 ניקוי נתוני בריאות חניך וסטטוס זוגיות

🆕 שדות שנוקו:
בעת מחיקת חונכות, המערכת כעת מנקה גם את:
  • tutee_wellness - נתוני בריאות החניך
  • relationship_status - סטטוס הקשר בין חונך לחניך

✅ הרציונל: שדות אלו צריכים להיות מלאים רק כאשר החונך בפעילות (חונכות פעילה). בעת סיום החונכות, הם צריכים להיות NULL.

לניקוי הנתונים הישנים בייצור:

⚠️ קבצי SQL חדשים:

cleanup_tutor_wellness_prod.sql

הריצת ניקוי הנתונים הישנים:

# ניקוי בטוח - מנקה רק אם סטטוס הוא "אין_חניך":

psql -h your_host -U your_user -d your_database -f childsmile/cleanup_tutor_wellness_prod.sql
✅ מה זה עושה:
  • מוצא את כל החונכים עם tutorship_status = 'אין_חניך'
  • מנקה את tutee_wellness ו-relationship_status עבורם
  • מעדכן את updated timestamp
  • בטוח: לא משפיע על חונכים בחונכות פעילה
