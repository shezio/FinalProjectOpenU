Why inject via DOM instead of JSX? The desktop .sidebar is hidden on mobile via display: none. Rather than restructuring the Sidebar component (which would risk breaking desktop), a vanilla DOM nav element is appended directly to document.body — outside React's tree — so it sits on top of everything and is not affected by the hidden sidebar's CSS.

Key insight: Every single rule in this file is wrapped inside @media (max-width: 767px). This means on a 768px+ screen (desktop/tablet) the entire file is ignored by the browser — it's physically impossible for these styles to affect the desktop layout.

The two most impactful rules:
1. .sidebar { display: none } + margin-right: 0 on content — removes the 280px sidebar and fills the freed space with the page content.
2. body { zoom: 1 !important } — overrides the zoom: 80% that common.css applies at 1800px and below, which would make everything tiny on a 390px phone screen.
